# Fase 001: agentes de experiência e valor

## Objetivo

Definir agentes que ajudem o Eat in Peace a melhorar a experiência dos consumidores do produto: restaurantes, gestores, donos de aplicações parceiras e clientes finais.

## Escopo

- Diferenciar consumidores B2B e cliente final.
- Definir agentes para adoção, embedding, confiança do cliente e descoberta de valor.
- Registrar o filtro de valor sem aumento de escopo.
- Atualizar docs comerciais e mapa de agentes.

## Testes

Esta fase não cria código executável. A verificação aplicável é documental:

- Agentes definidos com missão, usuários, entradas, saídas e fora de escopo.
- Valor adicional descrito com filtro explícito de não aumentar escopo.
- Documentos centrais atualizados para apontar para a nova camada.

Quando estes agentes virarem código, cada comportamento implementado deve ter testes unitários, testes de integração quando houver fronteira externa e teste end-to-end para o fluxo afetado.
