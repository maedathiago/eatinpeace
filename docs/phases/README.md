# Fases do projeto

Cada fase deve ter uma branch própria no formato `phase/<numero>-<slug>` e um documento nesta pasta.

| Fase | Branch | Objetivo |
| --- | --- | --- |
| [000: regras do repositório](000-repo-rules.md) | `phase/000-repo-rules` | Estabelecer branch por fase, stack Go + Supabase, documentação obrigatória e testes obrigatórios |
| [001: agentes de experiência e valor](001-experience-value-agents.md) | `phase/001-experience-value-agents` | Definir agentes para restaurantes, donos de aplicações parceiras e clientes finais, com valor adicional sem aumento de escopo |
| [002: squash antes de merge](002-squash-before-merge.md) | `phase/002-squash-before-merge` | Exigir commit único de fase antes do merge quando houver vários commits de trabalho |
